VerifClient

TEC file verification tool. It verifies card against customer specification.

Below is quick guide for some basic functionalities.

Verify card against customer specification
Manually created CSV

  • Create CSV file (in predefined format) from customer specification.
  • File > Load CSV
  • Browse for .csv file and click Open
  • Selected CSV will be indicated in the window bar.
  • (Optional) Tools > Edit Literals for adjusting labels of EF structure, types, and access conditions as per the CSV.
  • (Optional) Tools > Custom APDU if some APDU parameters need to be changed.
  • Configure Options and Security Codes to match card profile.
  • Tools > Run and click OK on confirmation dialog.
  • Verification will then proceed and result or errors (if any) will be displayed under Error Report tab.

Verify card against SIM Alliance's SIMPML
(.uxp)

  • File > Import From > SIMPML
  • Browse for .uxp file and click Open
  • Selected UXP will be converted to CSV and indicated in window bar as IMPORTED.
  • (Optional) Tools > Custom APDU if some APDU parameters need to be changed.
  • Configure Options and Security Codes to match card profile.
  • Tools > Run and click OK on confirmation dialog.
  • Verification will then proceed and result or errors (if any) will be displayed under Error Report tab.

Verify card against ex-Morpho SCDL report
exported from Card Designer

  • File > Import From > ex-Morpho XML
  • Browse for .xml file and click Open
  • Selected XML will be converted to CSV and indicated in window bar as IMPORTED.
  • (Optional) Tools > Custom APDU if some APDU parameters need to be changed.
  • Configure Options and Security Codes to match card profile.
  • Tools > Run and click OK on confirmation dialog.
  • Verification will then proceed and result or errors (if any) will be displayed under Error Report tab.
NOTE: import may fail if the XML generated from Card Designer was manually edited by hand with some weird characters added; raising encoding exception.
